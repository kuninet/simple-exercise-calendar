# エクササイズカレンダー テストシナリオ

## 基本機能テスト

### 1. アプリケーション起動テスト
- [ ] ブラウザでhttp://localhost:3000にアクセス
- [ ] カレンダーが正常に表示される
- [ ] 今日の日付がハイライトされている
- [ ] ユーザー選択ドロップダウンが表示される

### 2. ユーザー管理テスト
- [ ] ユーザー名をクリックしてドロップダウンを開く
- [ ] 3人のユーザーが表示される
- [ ] 異なるユーザーを選択する
- [ ] 統計データが切り替わる
- [ ] カレンダーデータが切り替わる

### 3. エクササイズ記録テスト
- [ ] エクササイズを選択する
- [ ] 「今日やった！」ボタンをクリック
- [ ] 褒めメッセージが表示される
- [ ] カレンダーの今日の日付が緑色になる
- [ ] 統計が更新される

### 4. 連続記録テスト
- [ ] 複数日連続でエクササイズを記録
- [ ] 連続記録メッセージが表示される
- [ ] 連続記録日に炎のアイコンが表示される
- [ ] 統計の連続記録数が正しく表示される

### 5. マイルストーンテスト
- [ ] 特定の記録数に達する（10回、30回など）
- [ ] マイルストーンメッセージが表示される
- [ ] 特別なアニメーションが再生される
- [ ] 紙吹雪エフェクトが表示される

### 6. カレンダーナビゲーションテスト
- [ ] 前月ボタンをクリック
- [ ] カレンダーが前月に移動する
- [ ] 次月ボタンをクリック
- [ ] カレンダーが次月に移動する
- [ ] 過去の記録が正しく表示される

### 7. 統計表示テスト
- [ ] 個人統計が正しく表示される
- [ ] 家族統計が正しく表示される
- [ ] ユーザー切り替え時に統計が更新される
- [ ] 記録後に統計が即座に更新される

### 8. データ管理テスト
- [ ] バックアップボタンをクリック
- [ ] 成功メッセージが表示される
- [ ] エクスポートボタンをクリック
- [ ] JSONファイルがダウンロードされる

### 9. エラーハンドリングテスト
- [ ] エクササイズ未選択で記録を試行
- [ ] 適切なエラーメッセージが表示される
- [ ] 重複記録を試行
- [ ] 「既に記録済み」メッセージが表示される

### 10. レスポンシブデザインテスト
- [ ] ブラウザウィンドウを縮小
- [ ] モバイル表示に切り替わる
- [ ] 統計が2列表示になる
- [ ] ユーザー選択が中央に配置される

## パフォーマンステスト

### 1. 読み込み速度
- [ ] 初回読み込みが3秒以内
- [ ] ユーザー切り替えが1秒以内
- [ ] 記録処理が2秒以内

### 2. メモリ使用量
- [ ] 長時間使用してもメモリリークなし
- [ ] アニメーション後のクリーンアップ確認

## ユーザビリティテスト

### 1. 直感性

- [ ] ボタンの意味が明確
